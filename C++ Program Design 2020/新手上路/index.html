    新手上路#

    一、环境的配置：#

    C++代码的编写需要一个顺手的编程环境

    请不要用记事本编写代码！！

    建议大家使用集成开发环境(IDE)或者可高度定制化的编辑器，可以帮忙大家在学习的前期暂时忽略掉一些琐碎的、恼人的细节，仍然可以写出代码并运行它

    • Windows：
      • Dev-C++：非常小巧的IDE，较为简陋，但安装即用
      • Codeblocks：很简洁的IDE，比Dev-C++更现代一点（请注意，建议大家下载自带编译器的版本，最好不要下载不带编译器的版本）
      • Visual Studio Code：功能强大的编辑器，可以通过安装插件来定制化，实现与IDE类似的功能（需要自己配置）
    • macOS：我其实不太了解，就在这里列举一下常见的orz
      • CLion
      • Visual Studio Code
      • Xcode
    • Linux 相信使用Linux系统的同学已经不需要我来推荐开发环境了（

    二、良好的编码习惯#

    即便是新手，也应该培养良好的编码习惯

    1. 命名习惯#

    尽管对变量的命名不会对程序的正确性造成任何影响，但我强烈建议你对变量的命名能够体现出其实际的作用

    一些例子：

    • 循环用变量：使用i, j, k……
    • 表示求和的结果：sum
    • 表示临时变量：tmp/temp
    • 表示分数：score
    • 表示老师的姓名：teacher_name/teacherName
    • 表示全体老师姓名的集合：teacher_names/all_teacher_name……

    2. 排版习惯#

    尽管代码的排版不会对程序的正确性造成任何影响，但我依然强烈建议你采用良好美观的布局（不然我阅读你的代码时会非常狂躁（雾））

    不用担心，上面我给出的编程环境都会提供"自动格式化"的功能，只需要一个快捷键，就可以自动把你的代码修改为美观的格式。

    3.注释习惯#

    尽管注释不会对程序的正确性造成任何影响，但我依然强烈建议你编写详细的注释

    注释的作用：解释你在做什么

    注释的作用对象：

    • 其他阅读你代码的人
    • 未来的你自己（不要相信自己一周后一定能读懂自己一周前的代码）

    4.抽象的习惯#

    每当你发现一些地方使用了相同的或者非常相似的逻辑时，尽量把这部分逻辑抽象为组件（函数、类等），而不是复制粘贴代码

    如果你之前没有太多编程经验，并不知道这部分在说什么，不要担心，以后我们会慢慢接触类似的思想

    如果你真的遇到了无法独立解决的问题，建议你遵循以下的几条建议

    我想这部分并不需要我过多解释

    交流不是抄袭！

    3. 求助学长学姐、助教、老师等#

    当你需要直接向上述的人求助的时候，我有以下建议给你：

    • 请保持礼貌

    • 准确描述你的问题，并使描述尽可能简短

    • 把代码提供给对方时，尽量不要截图或者直接把代码复制进微信。你可以将源文件发给对方，或者通过https://paste.ubuntu.com/ 等方式将代码上传，然后把链接发给对方
    • 你的代码尽量要满足前文提及的编码习惯
    • 不要过多依赖这个求助方式，尽量自己找到答案；但同时，也不要过于死磕一个问题，真的需要求助的时候，求助并不可耻。

    四、怎么学好C++#

    • 你需要多看一些标准的、准确的材料，包括老师的讲义、推荐教材以及互联网资料。 https://zh.cppreference.com/w/ https://en.cppreference.com/w/
    • 你需要多写代码，多练习，多实验。 当你不知道一段代码会发生什么时，把它写下来，运行它，得到结论。 ps：C++中有一个概念叫做“未定义行为”，含有未定义行为的代码的运行结果是实现相关的（也即是说，在你的电脑上是一种结果，在另一个人的电脑上可能就是另一种结果了），因此，也不能盲目相信自己实验的结论
    • 你需要多思考，有时可以复盘自己之前写的代码，看看有没有优化的余地。
